AlignMail — Estandarización y optimización de plantillas HTML de correo

Proyecto enfocado en alinear el diseño, código y mantenimiento de todas las plantillas de correo electrónico de la organización, mejorando la consistencia visual, el rendimiento y la autonomía de los equipos.

Contexto y objetivo

Los equipos de producto y marketing utilizaban múltiples plantillas de correo electrónico, desarrolladas de forma independiente para distintas aplicaciones. Esto provocaba inconsistencias en la identidad visual, errores en la maquetación y pérdida de tiempo en cada actualización.

El objetivo fue crear un sistema unificado de plantillas HTML con cabeceras y pies estandarizados, gestionado desde un entorno centralizado (SendGrid), garantizando una actualización ágil y una experiencia coherente para todos los productos.

Problema

Las plantillas existentes presentaban inconsistencias de marca, errores visuales y una falta de trazabilidad que ralentizaba la publicación de campañas y notificaciones automáticas.

Investigación

Se recopilaron y auditaron todas las plantillas utilizadas en los distintos productos. Se documentaron estilos, estructuras HTML y puntos de mejora, identificando redundancias y errores comunes.

  • Inconsistencia de tipografía y colores corporativos.
  • Cabeceras y pies con versiones distintas.
  • Ausencia de control de versiones o IDs únicos en SendGrid.

Definición

Se definieron los principales tipos de comunicación (transaccionales, promocionales, informativos) y se estableció un estándar visual y técnico para todas las plantillas. Se codificaron en HTML y se vincularon con SendGrid para trazabilidad y gestión modular.

Ideación y diseño

Se diseñó un sistema visual modular, documentando componentes reutilizables:

  • Botones, enlaces y espaciadores homogéneos.
  • Compatibilidad con Light y Dark Mode.
  • Soporte para distintos clientes de correo.

Los prototipos se validaron con el equipo de desarrollo y marketing mediante revisiones iterativas.

Unificación de branding

El proyecto permitió reducir drásticamente los tiempos de actualización y eliminar inconsistencias visuales entre productos. Marketing, diseño y desarrollo trabajan ahora con un mismo sistema validado y mantenible.

Lecciones aprendidas

  • Documentar el HTML base acelera la colaboración entre diseño y desarrollo.
  • El uso de IDs únicos en SendGrid facilita la trazabilidad.
  • Los sistemas modulares reducen la deuda técnica y mejoran la escalabilidad.
